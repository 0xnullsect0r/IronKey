Interface Layout

IronKey uses a three-panel PaneGrid layout managed by the iced 0.14 framework. The left panel shows connected drives and block devices, the centre panel provides a full file browser, and the right panel embeds a live PTY terminal. Panels can be resized by dragging the dividers between them. Focus a panel by clicking anywhere inside it or by pressing Tab to cycle between panels.

ℹ️
Kiosk vs Desktop

IronKey runs fullscreen in a Wayland kiosk (cage). On a desktop build, the window is resizable and decorations are enabled.

Drives Panel

The Drives panel enumerates all block devices by reading /sys/block/. Each drive shows its model, size, and partitions. Click a partition to select it; the action buttons below respond to the selection.

Operations

Button Description
Mount Mounts the selected partition. If the filesystem is LUKS-encrypted a passphrase prompt appears. NTFS and APFS are also handled automatically.
Unmount Safely unmounts the currently mounted partition.
Format Opens the format dialog. Choose filesystem type (ext4, FAT32, exFAT, NTFS, btrfs) and confirm by typing the device name.
Clone Copies the raw device to a .img file on another mounted path using a block-level copy.
Wipe Securely overwrites the partition with zeros or random data. Choose Fast (zeros) or Secure (random, multiple passes).
Info Opens a modal with full partition details and SMART health data (if available).
New Table Creates a new GPT or MBR partition table on the selected drive (destructive — requires typing the device name to confirm).

SMART Diagnostics

Select a partition and click Info to open the SMART panel. IronKey reads ATA SMART attributes via ioctl directly from the kernel — no smartmontools required.

The following attributes are displayed:

  • Overall health status
  • Temperature
  • Power-on hours
  • Reallocated sector count
  • Pending sector count
  • Uncorrectable sector count
  • Read error rate

File Browser Panel

The File Browser shows the contents of any mounted path. Navigate with single clicks; the path bar at the top shows your current location.

File Operations

Button Description
Copy (Ctrl+C) Marks selected files for copy.
Cut (Ctrl+X) Marks selected files for move.
Paste (Ctrl+V) Executes the pending copy or move into the current directory.
Rename (F2) Renames the selected file or folder inline.
Delete (Del) Moves the selected item to a temporary trash or deletes permanently (confirmation required).
Properties Shows a modal with file size, permissions, timestamps, and MIME type.

Search

Press Ctrl+F or click the search icon to open the search bar. IronKey recursively walks the current directory tree using walkdir and matches by filename or content substring.

ℹ️
Search scope

Search indexes the current directory at the time of query. For large drives, narrow your search by navigating into a subdirectory first.

File Viewer

Double-click any file to open it in the built-in viewer. IronKey detects the MIME type and renders:

  • Text / code files — syntax-highlighted (bash, Python, Rust, JSON, YAML, etc.)
  • Images — displayed inline (PNG, JPEG, GIF, BMP, WebP)
  • Binary files — shown as a hex dump

Terminal Panel

The Terminal panel embeds a full PTY terminal using portable-pty. It spawns zsh as the default shell with the IronKey environment loaded.

Shell Environment

  • Default shell: zsh with starship prompt
  • All 60+ pre-installed tools available on $PATH
  • Rust-native tools aliased over their classic counterparts
  • EDITOR=micro, PAGER=less, TERM=xterm-256color

ik-mount Helper

The ik-mount shell function provides a quick way to mount drives from the terminal:

bash 
# Mount a drive (prompts for passphrase if LUKS)
ik-mount /dev/sdb1

# Mount to a specific path
ik-mount /dev/sdb1 /mnt/recovery
ℹ️
Shared state

The terminal and the Drives panel share state — mounting a drive in the terminal will reflect in the Drives panel after clicking Refresh.

Modals Reference

Modal Trigger Description
Passphrase Mount on a LUKS partition Enter the LUKS decryption passphrase. Input is masked.
Format Format button Choose filesystem type, confirm by typing the device name (e.g. sda1).
Progress Clone / Wipe operations Shows live progress bar, transfer speed, and ETA.
Info / SMART Info button Partition details and SMART health table.
File Viewer Double-click a file Rendered file content (text, hex, image).
Properties Properties button File size, permissions, owner, timestamps, MIME type.
Shutdown Shutdown button Confirm graceful system shutdown.
Reboot Reboot button Confirm system reboot.
New Table New Table button Creates a new partition table. Requires typing the device name.

Keyboard Shortcuts

Shortcut Action
Tab Cycle focus between panels
Ctrl+C Copy selected file(s)
Ctrl+X Cut selected file(s)
Ctrl+V Paste
F2 Rename selected file
Del Delete selected file
Ctrl+F Open search
Ctrl+R / F5 Refresh drives and browser
Escape Close modal / cancel operation
