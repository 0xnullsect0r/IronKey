Build from Source

This section covers building and running IronKey directly on a Linux desktop for development and testing purposes. It does not produce a bootable USB image — see Build the Bootable ISO for that.

Prerequisites

  • Rust stable toolchain (via rustup)
  • Linux (X11 or Wayland)
  • libGL (typically provided by your GPU driver or mesa)

Install Rust

shell 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"

Clone and Build

shell 
git clone https://github.com/0xnullsect0r/IronKey.git
cd IronKey
cargo build -p ironkey-app --release

Run

shell 
./target/release/ironkey
ℹ️
Note

Running outside the bootable image means the GUI launches as a regular window. The Drives panel requires root to enumerate block devices.

Build the Bootable ISO

To create a proper bootable USB image you need to run the included build script, which automates bootstrapping a minimal Debian root filesystem, compiling the IronKey binary, and assembling a hybrid BIOS/UEFI ISO.

Host Requirements

⚠️
Root required on Debian / Ubuntu

The ISO build script must be run as root on a Debian or Ubuntu host. It uses debootstrap to bootstrap a minimal Debian bookworm rootfs.

Requirement Details
Host OS Debian 12 / Ubuntu 22.04+
Privileges Must be run as root (sudo)
Disk space ≥ 4 GB free in /tmp
Packages debootstrap squashfs-tools grub-pc-bin grub-efi-amd64-bin xorriso mtools
Rust toolchain rustup with stable toolchain

Install the required packages:

shell 
sudo apt install debootstrap squashfs-tools grub-pc-bin grub-efi-amd64-bin xorriso mtools

Build Steps

ℹ️
One command

The build script automates all 7 phases. Running sudo bash os/build.sh ironkey.iso is all you need.

shell 
sudo bash os/build.sh ironkey.iso

The script performs the following 7 phases in sequence:

  1. Compile binary — cargo build -p ironkey-app --release produces a static-linked ironkey binary.
  2. Bootstrap rootfs — debootstrap --variant=minbase bookworm creates a minimal Debian root filesystem with systemd, cage, zsh, and udev.
  3. Configure rootfs — Copies the IronKey binary, shell configs, starship config, and enables the ironkey.service systemd unit.
  4. Pack squashfs — Compresses the rootfs with mksquashfs -comp zstd -Xcompression-level 19 for maximum compression.
  5. Copy kernel — Extracts vmlinuz and initramfs.img from the bootstrapped rootfs.
  6. Set up GRUB — Copies the GRUB config for the bootloader menu.
  7. Build hybrid ISO — grub-mkrescue produces a hybrid ISO/USB image bootable from both BIOS and UEFI.
✅
Build complete

On success the ISO is written to the path you specified (e.g. ironkey.iso). A SHA-256 checksum is also printed.

Flash to USB

🚨
Data loss warning

Flashing will permanently erase all data on the target drive. Double-check the device path before continuing.

Use the included write-usb.sh helper:

shell 
# Identify your USB drive
lsblk

# Flash using the write-usb.sh helper
sudo ./write-usb.sh --drive /dev/sdX --iso ironkey.iso

Or manually with dd:

shell 
sudo dd if=ironkey.iso of=/dev/sdX bs=4M status=progress conv=fsync
sync

Download and install Rufus from https://rufus.ie, then follow these steps:

  1. Open Rufus and select your USB drive from the Device dropdown.
  2. Click SELECT and choose your ironkey.iso file.
  3. Leave Partition scheme as MBR or GPT depending on your target machine's firmware.
  4. Click START and confirm the warning.
  5. Wait for the operation to complete, then safely eject the drive.

Using dd:

shell 
# Find your USB disk number
diskutil list

# Unmount the disk (replace N with your disk number)
diskutil unmountDisk /dev/diskN

# Flash the ISO
sudo dd if=ironkey.iso of=/dev/rdiskN bs=4m
sync

Or use balenaEtcher (https://etcher.balena.io) for a GUI option.

Booting IronKey

  1. Insert the USB drive into the target machine.
  2. Enter BIOS/UEFI setup (usually F2, F12, Del, or Esc at boot).
  3. Set the USB drive as the first boot device.
  4. Save and exit — IronKey will boot directly into the GUI.
ℹ️
Silent boot

IronKey boots silently (kernel log level 0) directly into the fullscreen GUI via the cage Wayland kiosk compositor. No login prompt, no desktop environment.

⚠️
Disable Secure Boot

Secure Boot must be disabled in UEFI firmware settings. IronKey's GRUB image is not signed with a Microsoft-trusted key.

What you'll see on boot

After a few seconds the IronKey GUI fills the screen: Drives panel on the left, File Browser in the center, Terminal on the right. The Drives panel automatically enumerates all detected block devices.
